# Jarvis — Python Voice Assistant

A voice-activated desktop assistant inspired by Iron Man's J.A.R.V.I.S. It listens for the wake word **"Jarvis"**, then responds to spoken commands — opening websites, playing music, reading the latest news headlines, and answering general questions using OpenAI.

## Features

- 🎙️ **Wake-word activation** — continuously listens and springs to life on hearing *"Jarvis"*.
- 🌐 **Open websites** — Google, Facebook, YouTube, and LinkedIn by voice.
- 🎵 **Play music** — say `play <song>` to open a track from your personal music library.
- 📰 **News briefing** — reads out the top headlines (via NewsAPI).
- 🤖 **AI conversation** — falls back to OpenAI (GPT-3.5-turbo) for anything else, with spoken replies.
- 🔊 **Text-to-speech** — uses Google Text-to-Speech (gTTS) + Pygame for natural-sounding audio.

## Project Structure

| File | Purpose |
|------|---------|
| `main.py` | The main assistant — speech recognition, command routing, TTS, and the wake-word loop. |
| `musicLibrary.py` | A dictionary mapping song names to YouTube links used by the `play` command. |
| `client.py` | A standalone example of calling the OpenAI chat API (for testing/reference). |
| `temp.mp3` | Temporary audio file generated by gTTS during playback (auto-created and removed). |

## Requirements

- Python 3.8+
- A working microphone
- An [OpenAI API key](https://platform.openai.com/api-keys)
- A [NewsAPI key](https://newsapi.org/) (for the news feature)

### Python packages

```bash
pip install SpeechRecognition pyttsx3 requests openai gTTS pygame pyaudio
```

> **Note:** `SpeechRecognition` needs `pyaudio` for microphone input. On some systems you may also need `pip install pocketsphinx`.

## Setup

1. **Clone / download** this project.
2. **Add your API keys.** Open `main.py` and replace the placeholders:
   ```python
   newsapi = "<Your Key Here>"          # line 15 — your NewsAPI key
   ...
   client = OpenAI(api_key="<Your Key Here>")   # line 42 — your OpenAI key
   ```
   > 🔒 **Security tip:** Don't commit real API keys. Prefer environment variables, e.g.
   > `api_key=os.getenv("OPENAI_API_KEY")`.
3. **Customize your music library** (optional). Edit `musicLibrary.py` to add your own songs:
   ```python
   music = {
       "stealth": "https://www.youtube.com/watch?v=U47Tr9BB_wE",
       # "yoursong": "https://youtube.com/...",
   }
   ```

## Usage

Run the assistant:

```bash
python main.py
```

You'll hear *"Initializing Jarvis…"*. Then:

1. Say **"Jarvis"** — it replies *"Ya"* and starts listening for a command.
2. Speak a command:

| Say… | Jarvis does… |
|------|--------------|
| "open google" | Opens google.com |
| "open youtube" | Opens youtube.com |
| "open facebook" | Opens facebook.com |
| "open linkedin" | Opens linkedin.com |
| "play stealth" | Opens the *stealth* track from your music library |
| "tell me the news" | Reads out top Indian headlines |
| *anything else* | Sends it to OpenAI and speaks the reply |

## How It Works

1. **Listen** — `speech_recognition` captures microphone audio and transcribes it with Google's speech recognition.
2. **Wake word** — if the transcription equals `"jarvis"`, the assistant acknowledges and records the next command.
3. **Route** — `processCommand()` matches keywords (open/play/news) or otherwise forwards the request to OpenAI via `aiProcess()`.
4. **Speak** — responses are converted to speech with gTTS, saved to `temp.mp3`, and played through Pygame.

## Troubleshooting

- **No microphone found / PyAudio errors** — ensure `pyaudio` is installed and your mic is connected. On Windows, `pip install pipwin && pipwin install pyaudio` can help.
- **News not working** — verify your NewsAPI key and that you haven't hit the free-tier request limit.
- **OpenAI errors** — confirm your API key is valid and your account has quota.
- **Not recognizing "Jarvis"** — speak clearly in a quiet environment; recognition depends on internet access to Google's API.
